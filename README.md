# Secure Three-Tier AWS SaaS Architecture

This repository contains the final architecture project completed for my **AWS Cloud Architecture II** course while earning my **Bachelor of Applied Science in DevOps Engineering**.

The objective of this project was to design a secure, highly available AWS architecture for a SaaS application responsible for handling personally identifiable information (PII). Rather than focusing solely on implementation, the project emphasizes architectural decision-making, security, resiliency, automation, disaster recovery, and operational best practices.

## Project Objectives

- Design a secure, scalable three-tier AWS architecture
- Protect sensitive customer data using AWS security best practices
- Build for high availability and fault tolerance
- Incorporate DevSecOps principles into the overall design
- Consider monitoring, logging, disaster recovery, and operational readiness
- Evaluate architectural tradeoffs while balancing security, reliability, and cost
